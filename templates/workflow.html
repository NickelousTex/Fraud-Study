Overview of development of Fraud Detector

Data processing and cleaning

  • Identified and prioritized important variables
  • 22 features selected for initial analysis
  • Processed data for analysis

Model selection

  • Process:
    • out-of-box-model
    • cross-validated gridsearch
    • assessment of model using recall
  • Models tested: SVM, Logistic, GrandientBoostedClassifier, RandomForest, AdaBoost
  • Winner: GradientBoostedClassifier with 67% recall

Deployment

  • Flask framework used to tie python model with web
  • Bootstrap HTML/CSS layout used for visualization
  • MongoDB stored on an AWS EC2 instance
  • Data pulled in "real-time" through API
  • Dashboard shows latest probability

Future Work

  • Further process data and test features
  • Additional optimization of model to increase recall
  • Improve dashboard to show additional recent event cases
